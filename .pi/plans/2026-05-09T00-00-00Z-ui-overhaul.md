# UI overhaul to match design system

Bring the application's visual language in line with DESIGN.md: custom palette, Inter font,
card-on-background layout, pill buttons with the orange primary, and data-dense table styling.

## Goal

Every screen will reflect the DESIGN.md specifications — correct color tokens, Inter
typography, a gray-background + white-card page structure, pill-shaped buttons with the
coral orange primary, and tables with uppercase tracked headers and subtle row separators.
No decorative elements, no gradients, no zebra striping.

---

## Steps

### Step 1 — Design tokens and font

Replace the current Geist + zinc/neutral palette with the DESIGN.md tokens and Inter font.

- [ ] Swap the Google Fonts import in `app.css` from Geist to **Inter** (weights 400, 500, 600, 700)
- [ ] Rewrite the `:root` CSS variables in `app.css` to match the design palette:
  - `--background: #F5F6F8` (page), `--card: #FFFFFF` (surfaces)
  - `--primary: #FF6B35`, `--primary-foreground: #FFFFFF`, `--primary-hover: #E85A24`
  - `--border: #E8E9EB`, `--border-subtle: #F0F1F3`
  - `--foreground: #111827`, `--muted-foreground: #6B7280`, `--muted: #9CA3AF`
  - `--danger: #EF4444`
- [ ] Update `--font-sans` in the `@theme` block to `'Inter', ui-sans-serif, system-ui`
- [ ] Verify `npm run build` passes after token changes

---

### Step 2 — Page shell

Apply the card-on-background layout and tighten the header toolbar in `App.vue`.

- [ ] Set page wrapper to `bg-background` (now the light gray `#F5F6F8`)
- [ ] Wrap the main content in a white card: `bg-card border border-border rounded-lg shadow-card`
- [ ] Header toolbar: page title (`1.5rem / 700`) left-aligned, `CompanyDropdown` right-aligned, separated by `justify-between`
- [ ] Apply correct spacing: `px-6 py-5` inside the card header, `gap-0` between toolbar and tab content

---

### Step 3 — Buttons

Override shadcn's Button variants to match the pill shape and design palette.

- [ ] Set global `--radius` to produce `rounded-full` on buttons
  - Or override directly in `Button.vue` — add `rounded-full` to both `default` and `outline` variants
- [ ] Primary variant: `bg-[#FF6B35] hover:bg-[#E85A24] text-white font-semibold`
- [ ] Secondary / outline variant: `bg-white border-border text-foreground font-medium hover:bg-[#FAFAFA]`
- [ ] Apply correct padding: `px-4 py-2` (`8px 16px`), `text-sm`
- [ ] Confirm Submit and Add Row buttons reflect the new styles in `NewEntriesTab.vue`

---

### Step 4 — Table polish

Style both tables (New Entries + History) to match the data-dense specification.

- [ ] Table headers: `text-xs font-semibold uppercase tracking-wider text-muted-foreground`
- [ ] Table container: white card with `border border-border rounded-lg shadow-card overflow-hidden`
- [ ] Row borders: `border-b border-[#F0F1F3]` (border-subtle) — no zebra striping, no row shadows
- [ ] Row hover: `hover:bg-[#FAFAFA]` only — no color fills
- [ ] Cell padding: `px-4 py-3` (16px horizontal, 12px vertical)
- [ ] Apply consistently to `NewEntriesTab.vue` and `HistoryTab.vue`

---

### Step 5 — Inputs and combobox trigger

Align `Input` and `AppCombobox` trigger styling with the DESIGN.md input spec.

- [ ] Input: `border-border rounded-md text-sm px-3 py-2 placeholder:text-muted`
  - `bg-white` surface, `focus-visible:ring-primary/30`
- [ ] `AppCombobox` trigger button: match the same height and border style as `Input`
  - Replace the current `Button outline` trigger with a styled `div` or override to look like an input field (border, radius, bg-white, text-sm)
- [ ] Disabled state: `opacity-50 cursor-not-allowed` on both
- [ ] Confirm all comboboxes and inputs look visually consistent across `NewTimeEntryRow.vue`

---

## Notes

- All color values in steps 3–5 reference DESIGN.md tokens; prefer the CSS variable where it exists (`var(--primary)`) and fall back to the hex literal only when no semantic token maps cleanly.
- `--radius` is currently used by shadcn components globally; changing it to `9999px` would make all components pill-shaped. Since only buttons need `rounded-full` (inputs use `rounded-md`), it is safer to override per-variant in `Button.vue` rather than changing the global radius token.
- The DESIGN.md specifies no dark mode — the `.dark` block in `app.css` can be removed or left inert; it will not be used.
- Tag / badge styling (semantic colors for company names, task types) is out of scope for this plan — it can be addressed in a follow-up once the base tokens and layout are solid.
